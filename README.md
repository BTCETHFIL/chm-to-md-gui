# CHM to Markdown Converter (GUI)

> 一键将 CHM 帮助文档转换为结构化 Markdown 文件，保留完整目录树和层级编号。

[![Python](https://img.shields.io/badge/Python-3.7+-blue.svg)](https://www.python.org/)
[![License](https://img.shields.io/badge/License-MIT-green.svg)](LICENSE)
[![Platform](https://img.shields.io/badge/Platform-Windows-lightgrey.svg)](#)

---

## ✨ 功能特点

- **图形界面** — 拖拽式操作，支持单文件 / 多选 / 文件夹批量转换
- **目录树还原** — 解析 CHM 的 `.hhc` 目录文件，生成完整嵌套目录结构
- **层级编号** — 自动分配 `1.2.7.5` 格式的数字编号，目录和文件名都带编号
- **统一编号** — 多文件转换时可选择整体统一编号，跨文件检索无忧
- **递归折叠** — 智能展平深层 TOC 为两层结构，输出目录零嵌套
- **合并 CHM 支持** — 自动处理 `merge` 引用，合并主子 CHM 到统一目录树
- **图片内嵌** — 将图片转为 base64 内嵌，单文件即开即用
- **多种提取策略** — 7-Zip → hh.exe → ITSF 纯 Python 解析，三重兜底
- **智能表格对齐** — 自动计算中文字符宽度，对齐 Markdown 表格
- **编码检测** — 自动识别 GBK/UTF-8 等编码，正确处理中文内容
- **进度日志** — 实时显示转换进度和彩色日志

## 🚀 快速开始

### 安装依赖

```bash
pip install -r requirements.txt
```

### 启动程序

```bash
python chm_to_md_gui.py
```

或直接双击 `启动.bat`

### 基本使用

1. 选择模式：**单文件** / **多选** / **文件夹**
2. 点击 **浏览** 选择 `.chm` 文件或文件夹
3. 如检测到多个文件，弹出对话框选择是否整体统一编号
4. 选择输出目录
5. 点击 **开始转换**
6. 等待完成，在输出目录查看结果

## 📁 输出结构

### 单文件转换

```
输出目录/
└── 文档名/
    ├── 1 欢迎使用VBScript/
    │   ├── 1.1 VBScript 用户手册/
    │   │   ├── 1.1.1 什么是 VBScript？.md
    │   │   ├── 1.1.2 VBScript基础/
    │   │   │   ├── 1.1.2.1 VBScript 数据类型.md
    │   │   │   └── ...
    │   │   └── ...
    │   └── 1.2 VBScript 语言参考/
    │       └── ...
    └── ...
```

> 只包含转换后的 `.md` 文件，按 TOC 层级组织，目录名和文件名都带数字编号。无多余文件、无辅助输出。

### 多文件整体统一编号（选 [是]）

```
输出目录/
├── 01_ABCONTROLLOGIX驱动/           ← 两层结构：序号_驱动名
│   ├── ABCONTROLLOGIX驱动.md         ← 目录页
│   ├── 1.1.1 配置驱动.md
│   ├── 1.1.2 配置位号IO地址.md
│   └── ...
├── 02_BACnetIP驱动/
│   ├── BACnetIP驱动.md
│   ├── 2.1.1 协议介绍.md
│   └── ...
└── 12_InovanceTcpNet驱动/
    ├── InovanceTcpNet驱动.md
    ├── 12.1.1 概述.md
    └── ...
```

> 目录名格式：`序号_首条目标题`（如 `12_InovanceTcpNet驱动`）。<br>
> 递归折叠深层 TOC，无论原目录多深，输出始终为两层结构，零子目录嵌套。<br>
> 实测：46 个 CHM 含最深 8 层 TOC → 输出 46 个扁平目录、1514 个 .md 文件。<br>
> 文件名按字母排序分配书号，保证编号稳定可复现。

### 多文件独立编号（选 [否]）

```
输出目录/
├── 文档A/
│   └── 1 章节/
│       └── 1.1 内容.md
├── 文档B/
│   └── 1 章节/
│       └── 1.1 内容.md     ← 各自从1开始
└── ...
```

## 📋 目录编号规则

每个文件、目录名和 Markdown 标题都带有层级数字编号，便于快速识别：

| 层级 | 格式 | 示例 | 说明 |
|---|---|---|---|
| 单文件 | `N.N.N` | `1.2.7` → VBScript 函数 | 常规层级编号 |
| 统一编号 | `M.N.N.N` | `3.1.2.7` → 第3本书的1.2.7节 | 文件夹/多选模式选[是] |

> 编号同时出现在：① 目录路径 ② 文件名 ③ Markdown H1 标题，三重一致不漏。

## ⚙️ 提取策略

程序按以下优先级尝试提取 CHM 内容：

1. **7-Zip** (推荐) — 速度最快，支持最好
2. **hh.exe** — Windows 自带，无需额外安装
3. **ITSF 纯 Python** — 内置兜底，无任何外部依赖

## 📦 依赖

| 包 | 用途 |
|---|---|
| `beautifulsoup4>=4.12` | HTML 解析与清理 |
| `html2text>=2024.2` | HTML → Markdown 转换 |
| `pypinyin>=0.51` | 中文文件名转拼音 |
| `chardet>=5.2` | 编码自动检测 |
| `lxml>=5.1` | 快速 XML/HTML 解析（可选） |

> 可选：安装 [7-Zip](https://www.7-zip.org/) 获得最佳提取体验。

## 🛠️ 开发相关

### 调试模式

转换过程中会在系统临时目录保存调试文件（30分钟自动清理）：

- `_toc_flat.json` — 扁平化目录数据
- `_toc_map.json` — 路径映射统计
- `_convert_summary.json` — 转换摘要
- `_main.hhc` / `_sub_*.hhc` — 原始 TOC 数据

> 调试文件不会出现在输出目录，不影响转换结果。

### 灵感来源

本项目基于 [chy5301/chm-to-markdown-converter](https://github.com/chy5301/chm-to-markdown-converter) 重写，新增 GUI 界面、合并 CHM 支持、层级编号、图片内嵌等功能。
